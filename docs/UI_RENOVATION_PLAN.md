# HomeAutomationSystem UI/3D 改造规划书

## 目标
- 使用 `OBJ_loader` 加载 `resources/models/home.obj`，以白模方式在 Qt OpenGL 中渲染。
- 将主界面改为“模型为主、动效为辅”的交互布局。
- 保留并增强现有手势识别链路，将识别结果映射为模型交互动作。
- 提供机位快照系统（保存、读取、持久化）与光照系统（全局光 + 可开关矩形顶灯）。

## 架构拆分
1. 渲染层：`ModelOpenGLWidget`
- 基于 `QOpenGLWidget`。
- 内嵌 GLSL，支持基础白模光照。
- 支持全局方向光与顶灯近似面积光。
- 支持鼠标轨道相机交互。

2. 相机层：`CameraSystem`
- 机位参数：`position/target/up/fov`。
- 机位操作：保存、索引读取、名称读取、插值过渡。
- 序列化：`snapshotsToJson/snapshotsFromJson`。

3. 光照层：`LightingSystem`
- 全局光参数（方向、环境光、漫反射、高光）。
- 顶灯参数（位置、尺寸、颜色、强度、衰减、开关）。

4. UI 层：`MainWidget`
- 主区：3D 模型视口。
- 侧栏：设备序列帧、机位按钮、顶灯开关、实时状态文本。
- 调试面板：保留现有实时识别与骨架预览能力。

5. 交互映射（手势 -> UI/3D）
- `swipe_right`：切换设备序列 + 下一个机位。
- `point_left`：上一个设备序列 + 上一个机位。
- `point_right`：下一个机位。
- `open`：顶灯开启。
- `close`：顶灯关闭。
- `cheese`：保存当前机位快照。

## 风险与后续
- 当前面积光为近似实现（非物理真实矩形光），可后续升级为更高质量 BRDF/IBL 方案。
- 若后续引入阴影与后处理，需要补充 FBO/深度图渲染链。
