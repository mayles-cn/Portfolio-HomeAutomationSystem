# 智能家居手势控制系统 - 项目简历描述

## 项目名称

智能家居手势控制系统（HomeAutomationSystem）

## 一句话介绍

一个基于 Qt OpenGL 与 MediaPipe 手部追踪的智能家居 3D 可视化控制系统，通过实时手势识别驱动场景切换与设备操作。

## 项目简介

该项目面向"手势交互 + 智能家居可视化"场景，实现从摄像头实时采集手部关键点、手势分类推理、到 3D 家居场景联动控制的完整闭环。系统渲染 OBJ 格式的家居模型，支持 9 个预设相机机位对应 8 类家居设备（空调、洗衣机、油烟机、风扇、音响、窗帘、咖啡机、门锁），用户通过手势即可完成场景浏览、设备动画播放、灯光开关等操作，无需触碰任何物理控件。

## 技术栈

- C++17
- Qt 6（Widgets / OpenGL / OpenGLWidgets）
- CMake
- MediaPipe Hand Landmarker（手部关键点检测）
- 自训练轻量级 C++ 手势分类模型

## 核心模块

1. 3D 渲染模块：OBJ 模型加载与三角化、方向光 / 面积光 / 网格地面渲染、相机轨道控制与机位快照管理。
2. 手势识别模块：MediaPipe 桥接进程管理、126 维特征提取、滑动窗口序列推理、事件门控（置信度阈值 + 冷却机制）。
3. 场景联动模块：手势事件到相机机位 / 设备动画 / 灯光状态的映射与驱动。
4. 动画播放模块：帧序列图片加载与循环播放，8 套设备动画资源管理。
5. 调试面板模块：F9 热键唤出，实时骨架可视化、预测结果监控、手势屏蔽控制、渲染参数调节。

## 个人贡献亮点

1. 设计并实现手势识别全链路：MediaPipe 关键点采集 → 特征提取 → 模型推理 → 门控过滤 → 场景响应。
2. 实现应用层手势冷却机制（1200ms），解决高频手势连续触发导致的状态抖动问题。
3. 构建 9 机位 × 8 设备的场景映射体系，手势 / 键盘双通道驱动相机与设备动画同步切换。
4. 基于 OpenGL 自研 OBJ 加载器与多光源着色器，支持方向光、面积光、模型透明度等参数实时调节。
5. 设计无边框浮动覆盖层架构，实现设备信息提示与动画预览的非侵入式叠加显示。

## 项目结果

1. 形成可运行的 Windows 桌面端演示程序，启动即自动开始手势识别。
2. 支持 6 种手势操作：场景前进 / 后退、设备启动 / 停止、灯光切换。
3. 8 类家居设备具备独立帧序列动画，配合 3D 场景机位形成完整的可视化交互体验。
4. 内置调试面板，支持实时骨架绘制、手势屏蔽、渲染参数微调，便于开发迭代与现场演示。

## 可量化描述

1. 集成 MediaPipe 手部追踪 + 自训练分类模型，实现 8 类手势的实时识别与事件触发。
2. 管理 9 个相机机位与 8 套设备帧序列动画（共计 700+ 帧图片），构建完整的场景设备映射体系。
3. 通过门控机制（置信度阈值 + 连续帧确认 + 1200ms 冷却）将误触发率降低，保障交互稳定性。
4. 打通手势输入 → 模型推理 → 场景渲染 → 设备动画四条链路，构建端到端无接触交互闭环。
